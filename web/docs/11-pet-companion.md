# 桌宠功能说明

本文档说明 Ueat 当前原型中的桌宠功能范围、交互规则、活跃度投喂机制、素材来源与后续扩展计划。

## 功能定位

桌宠是 Ueat 的轻量陪伴与活跃度反馈系统。它不是独立游戏，而是挂在主应用界面上的互动角色：

- 用户约饭、发卡片、发帖、评论、点赞、收藏、转发、发消息等行为会给桌宠增加经验、饱食、心情或亲密度。
- 桌宠会通过动画和台词反馈用户的活跃行为。
- 第一版使用临时 VPet 动画资源验证交互体验，后续可替换为 Ueat 官方素材库和自定义形象系统。

## 当前入口

- 全局桌宠：用户登录并完成资料后，在主界面显示。
- 我的页面：提供桌宠管家入口，可重新显示桌宠、投喂桌宠和查看基础状态。
- 桌宠本体：显示在页面左下角附近，可拖动到屏幕任意位置。

## 交互规则

### 小人本体

- 拖动小人本体：移动桌宠位置。
- 轻触小人本体：触发摸头互动和台词，不打开状态面板。
- 小人位置会保存在本地，下次打开继续沿用。

### 状态条

小人底部的 `Lv / 心情` 状态条用于打开或关闭状态面板。

状态条使用浅色底和深色文字，避免和角色衣服、阴影或背景重叠时看不清。

### 侧边按钮

- 餐具：快捷投喂。
- 衣服：预留给后续官方素材库和换装系统，当前只给提示反馈。
- `-`：收起桌宠为小球。

### 小球形态

- 点击 `-` 后，桌宠收缩为浅黄色圆形小球。
- 点击小球可展开回正常桌宠。
- 小球也可以拖动。

### 状态面板

状态面板目前展示：

- 等级
- 经验
- 饱食
- 心情
- 亲密
- 大小切换
- 睡觉
- 爬墙

面板刻意保持小尺寸，避免遮挡主应用操作区域。素材来源不在面板中展示，统一放在本文档和资源 NOTICE 中说明。

## 活跃度投喂

当前通过 `web/src/lib/petActivity.ts` 分发桌宠活跃事件，通过 `web/src/hooks/usePetCompanion.ts` 统一计算奖励和每日上限。

| 行为 | 事件名 | 说明 |
| --- | --- | --- |
| 手动投喂 | `manual_feed` | 点击餐具按钮 |
| 发布约饭卡片 | `meal_card` | 成功发卡片后奖励 |
| 交换/约饭邀请 | `exchange` | 发起约饭邀请后奖励 |
| 发布帖子 | `post` | 成功发社区帖子后奖励 |
| 评论 | `comment` | 成功发表评论后奖励 |
| 点赞 | `like` | 首次点赞奖励 |
| 收藏 | `favorite` | 首次收藏奖励 |
| 转发 | `share` | 转发帖子后奖励 |
| 私聊消息 | `message` | 发送文本、图片或语音消息后奖励 |
| 群聊广场 | `group` | 创建或加入公开群聊后奖励 |

每种行为有每日奖励上限，避免无限刷经验。

## 状态与存储

桌宠状态保存在浏览器 `localStorage` 中，当前 key 为：

```text
ueat-pet-companion-v2
```

保存内容包括：

- 是否显示
- 是否收缩
- 等级与经验
- 饱食、心情、亲密
- 桌宠大小
- 屏幕位置
- 当日各类奖励次数

当前状态仍是前端本地状态，不参与账号云同步。后续若要多设备同步，需要在后端增加用户桌宠状态表。

## 当前素材来源

第一版桌宠临时使用 VPet 项目的部分内置动画资源，仅用于非商业课程原型验证。

来源：

```text
https://github.com/LorisYounger/VPet
```

本项目当前只复制了少量 PNG 帧资源，位置为：

```text
web/public/assets/vpet-prototype/frames/
```

资源说明文件：

```text
web/public/assets/vpet-prototype/NOTICE.md
```

注意：

- VPet 代码许可证和内置图片/动画素材要求不完全相同。
- 当前资源仅用于学校课程原型展示，不作为商业素材包分发。
- 若未来用于正式上线或商业发布，应替换为 Ueat 自有素材或确认完整授权。

## 后续官方素材库规划

正式版建议将桌宠系统拆成三层：

1. 官方素材库

   管理发型、瞳色、上衣、头饰、手持物、兴趣元素等素材。素材需要统一命名、分层、尺寸、锚点和动作适配规范。

2. 用户换装配置

   用户保存当前形象配置，例如：

   - hair
   - eyeColor
   - outfit
   - accessory
   - handheld
   - themeTag

3. 动作与行为系统

   不同状态和行为触发不同动画，例如：

   - idle
   - happy
   - touch
   - eat
   - sleep
   - levelUp
   - climb

## 个性化方向

后续可以根据用户资料和偏好标签生成初始形象，例如：

- 喜欢摄影：相机挂件或小相机手持物。
- 喜欢读书：书本头饰、书包或书签元素。
- 喜欢安静：柔和发色、低饱和服装。
- 喜欢运动：运动外套、发带或水杯。
- 喜欢探店：餐具、甜品、小地图元素。

这些元素不建议让用户从零绘制，而应从官方素材库中组合，保证风格统一和动画适配稳定。

## 实现文件索引

- `web/src/components/pet/PetCompanion.tsx`：桌宠 UI、拖动、面板、小球、按钮。
- `web/src/hooks/usePetCompanion.ts`：桌宠状态、奖励规则、本地存储。
- `web/src/components/pet/vpetFrames.ts`：动画帧清单。
- `web/src/lib/petActivity.ts`：跨页面活跃事件分发。
- `web/src/App.tsx`：全局挂载桌宠并接入发卡、发帖、评论、点赞等奖励。
- `web/src/pages/Profile.tsx`：我的页面桌宠管家入口。
- `web/src/components/chat/ChatDetail.tsx`：消息发送奖励。
- `web/src/components/chat/ConversationList.tsx`：群聊广场奖励。

## 当前限制

- 桌宠状态只保存在本地浏览器，不随账号同步。
- 当前动画是 PNG 帧序列，不是完整 Live2D 模型。
- 当前换装按钮只是预留入口，尚未接入官方素材库。
- 当前素材不作为正式商用资源。
- 桌宠与主应用布局仍需要继续做更多机型和屏幕尺寸适配。
