# 19 新用户轻量引导

本文记录 U eat 当前的新用户场景提示实现。目标不是做完整教程页，而是在用户第一次进入具体功能时，用轻量、可关闭的小提示说明关键动作。

## 设计原则

- 不使用全屏遮罩，不阻断首页、聊天、社区和我的页的主流程。
- 提示只解释当前屏幕上马上能做的动作，文案保持短句。
- 每条提示可手动关闭，也会在用户完成相关动作后自动记住。
- 提示按用户隔离存储；未登录或缺少用户 id 时使用 `guest` fallback。
- 样式沿用现有 Tailwind、圆角、半透明浅色面板和轻量阴影，避免营销感。

## 本地存储

提示状态由 `web/src/hooks/useOnboardingHints.ts` 管理：

```text
ueat-onboarding-hints-v1:<userId | guest>
```

值为已读提示 id 数组。当前没有设置页重置入口；后续如果加入“重看新手引导”，可以清除该 key 或提供按 id 重置。

## 提示矩阵

| id | 场景 | 触发 | 自动完成 |
| --- | --- | --- | --- |
| `home-swipe` | 首页约饭卡 | 有可浏览饭卡且未读 | 点击下一张、左滑换卡、右滑邀请或确认邀请 |
| `home-background` | 我的页背景入口 | 我的页账号卡片下方 | 打开背景选择器或关闭提示 |
| `chat-background` | 聊天设置 | 打开聊天/群聊设置 | 进入当前聊天背景选择器或关闭提示 |
| `pet-enable` | 我的页桌宠管家 | 桌宠隐藏且未读 | 点击打开桌宠或关闭提示 |
| `pet-feed-water` | 我的页桌宠管家 | 桌宠显示且没有更高优先级状态提示 | 投喂、喂水或关闭提示 |
| `pet-low-hunger` | 我的页桌宠管家 | 桌宠显示且饱食不高于 35 | 投喂或关闭提示 |
| `pet-low-mood` | 我的页桌宠管家 | 桌宠显示且心情不高于 40 | 喂水或关闭提示 |
| `pet-level-up` | 我的页桌宠管家 | 桌宠显示且等级大于 1 | 关闭提示 |
| `pet-edge` | 我的页桌宠管家 | 桌宠处于贴边或爬墙状态 | 关闭提示 |
| `wardrobe-intro` | 桌宠衣柜 | 首次打开衣柜 | 关闭提示 |

## 当前文案

首页约饭卡：

```text
左滑看下一张，右滑就是想一起吃。轻轻一滑就好，也可以点进卡片看详情。
```

背景：

```text
首页和我的主页共用这一张背景；每个聊天详情页可以单独设置自己的背景。
```

桌宠：

```text
桌宠默认关闭。打开后可以投喂、喂水和互动，发约饭卡、聊天、点赞、发帖都会带来成长。
有点饿了。投喂会恢复饱食，也会增加一点亲密度。饱食太低时桌宠会更安静。
心情需要照顾。喂水、摸摸、聊天和发动态都能让心情慢慢恢复。
投喂和喂水。这两个按钮会影响饱食、心情和亲密度，也是最轻量的升级方式。
桌宠升级。升级后会解锁更多动作反馈。继续约饭、互动和社区行为就会积累经验。
贴边和爬墙。A 款动态桌宠可能会贴边或爬墙，拖一下就能带回屏幕中间。
```

衣柜：

```text
A 是更活跃的动态桌宠；B 更像头像贴纸装扮。两款贴纸会分别保存，右侧贴纸可拖到画布，选中后可拖动、拉角缩放。
```

聊天背景：

```text
每个聊天详情页都能单独设置背景，不会覆盖首页和我的主页背景。
```

## 实现文件

- `web/src/hooks/useOnboardingHints.ts`：提示 id、localStorage 读写、按用户隔离的已读状态。
- `web/src/components/onboarding/OnboardingHint.tsx`：通用小提示 UI。
- `web/src/pages/Home.tsx`：首页约饭卡手势提示。
- `web/src/pages/Profile.tsx`：背景共享、桌宠开启、投喂/喂水、低饱食、低心情、升级、贴边/爬墙提示。
- `web/src/components/chat/ChatDetail.tsx`：聊天设置里的当前聊天背景提示。
- `web/src/components/pet/PetWardrobePage.tsx`：A/B 桌宠和贴纸操作提示。

## 后续可选增强

- 在设置页增加“重看新手引导”入口。
- 新注册用户完成资料页后，可只在第一次回到首页时强制显示 `home-swipe`，其余仍按场景触发。
- 如果未来把提示云同步，需要新增轻量用户偏好字段，避免设备间重复出现。
